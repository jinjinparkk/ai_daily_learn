오늘의 기초 · Fundamentals

단어 임베딩 Word Embeddings
이게 뭐야? · What is it
ENA word embedding is a way to represent a word as a list of numbers, called a vector.
KO단어 임베딩은 하나의 단어를 벡터라고 부르는 숫자들의 목록으로 표현하는 방법입니다.
ENThe key idea is that words with similar meanings end up with similar vectors.
KO핵심 아이디어는 비슷한 의미를 가진 단어들이 서로 비슷한 벡터를 갖게 된다는 것입니다.
ENThis lets a model measure how close two words are just by comparing their numbers.
KO덕분에 모델은 두 단어가 얼마나 가까운지를 숫자만 비교해서 측정할 수 있습니다.
작동 원리 · How it works
ENFirst, every word in the vocabulary is assigned a vector of, for example, 300 numbers.
KO먼저 어휘에 있는 각 단어에 예를 들어 300개의 숫자로 된 벡터가 배정됩니다.
ENAt the start these numbers are random and carry no meaning.
KO처음에는 이 숫자들이 무작위이며 아무 의미도 담고 있지 않습니다.
ENThe model then reads huge amounts of text and learns which words appear near each other.
KO그다음 모델은 방대한 양의 텍스트를 읽으며 어떤 단어들이 서로 가까이 등장하는지를 학습합니다.
ENIt gradually adjusts the vectors so that words used in similar contexts move closer together.
KO모델은 비슷한 문맥에서 쓰이는 단어들이 서로 가까워지도록 벡터를 조금씩 조정합니다.
ENAs a result, 'king' and 'queen' land near each other, while 'king' and 'banana' stay far apart.
KO그 결과 'king'과 'queen'은 서로 가까이 위치하고, 'king'과 'banana'는 멀리 떨어지게 됩니다.
ENThese learned vectors then become the input that larger models build on for tasks like translation.
KO이렇게 학습된 벡터는 번역 같은 작업을 수행하는 더 큰 모델의 입력이 됩니다.
🧩 비유로 이해 · Analogy
ENThink of a giant map where each word is a city, and cities with related meanings are built close together.
KO각 단어가 하나의 도시인 거대한 지도를 상상해 보세요. 의미가 관련된 도시들은 서로 가까이 세워집니다.
기본 용어 · Key terms
Vector (벡터)숫자들을 나열한 목록으로, 단어의 의미를 수치로 나타낸 것.
Vocabulary (어휘)모델이 알고 있는 단어 또는 토큰의 전체 집합.
Context (문맥)어떤 단어 주위에 나타나는 다른 단어들, 의미를 파악하는 단서.
Similarity (유사도)두 벡터가 얼마나 가까운지를 나타내는 척도.

오늘의 핵심 개념 · Key Concept

토크나이저 확장 Tokenizer Expansion
ENA tokenizer splits text into small pieces called tokens before a model can process it.
KO토크나이저는 모델이 처리하기 전에 텍스트를 토큰이라는 작은 조각으로 나눕니다.
ENThe set of tokens is usually fixed at the very start of training and rarely changes afterward.
KO토큰 집합은 보통 학습 맨 처음에 고정되며 이후에는 거의 바뀌지 않습니다.
ENIf a language is added later, its words get chopped into many tokens, which slows the model down.
KO나중에 어떤 언어가 추가되면 그 단어들이 여러 토큰으로 잘게 쪼개져 모델이 느려집니다.
ENTokenizer expansion updates the token set in place so most old tokens stay the same while new ones are added.
KO토크나이저 확장은 대부분의 기존 토큰을 그대로 두면서 새 토큰을 더해 토큰 집합을 그 자리에서 갱신합니다.
ENNew token vectors are initialized as the average of their sub-parts, giving them a sensible starting meaning.
KO새 토큰의 벡터는 그 하위 조각들의 평균으로 초기화되어 합리적인 시작 의미를 갖게 됩니다.
왜 지금? · Why now
ENAs on-device and multilingual models grow, cheaply upgrading a fixed tokenizer is becoming a practical need.
KO기기 내장형·다국어 모델이 늘어나면서, 고정된 토크나이저를 저렴하게 업그레이드하는 일이 실질적인 필요가 되고 있습니다.
