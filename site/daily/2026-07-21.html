오늘의 기초 · Fundamentals

어텐션 메커니즘 Attention Mechanism
이게 뭐야? · What is it
ENThe attention mechanism lets a model decide which parts of the input matter most for each output it produces.
KO어텐션 메커니즘은 모델이 각 출력을 만들 때 입력의 어느 부분이 가장 중요한지 스스로 판단하게 해 줍니다.
ENIt is the core idea behind the Transformer, the architecture that powers modern language and vision models.
KO이는 현대 언어·비전 모델의 근간이 되는 아키텍처인 트랜스포머의 핵심 아이디어입니다.
ENInstead of treating every word or pixel equally, attention assigns different weights based on relevance.
KO모든 단어나 픽셀을 동등하게 취급하는 대신, 어텐션은 관련성에 따라 서로 다른 가중치를 부여합니다.
작동 원리 · How it works
ENEach input token is turned into three vectors: a query, a key, and a value.
KO각 입력 토큰은 쿼리, 키, 값이라는 세 가지 벡터로 변환됩니다.
ENTo decide how much one token should focus on another, the model compares that token's query with the other's key.
KO한 토큰이 다른 토큰에 얼마나 집중할지 결정하기 위해, 모델은 그 토큰의 쿼리를 상대의 키와 비교합니다.
ENA high similarity score means the model pays more attention to that token.
KO유사도 점수가 높다는 것은 모델이 그 토큰에 더 많은 주의를 기울인다는 뜻입니다.
ENThese scores are passed through a softmax function so they add up to one, forming attention weights.
KO이 점수들은 소프트맥스 함수를 거쳐 합이 1이 되도록 정규화되어 어텐션 가중치가 됩니다.
ENThe output is a weighted sum of all the value vectors, blending information from the most relevant tokens.
KO최종 출력은 모든 값 벡터의 가중합으로, 가장 관련 있는 토큰들의 정보를 섞어 만듭니다.
ENBecause every token can look at every other token, attention captures long-range relationships that older models struggled with.
KO모든 토큰이 다른 모든 토큰을 볼 수 있기 때문에, 어텐션은 과거 모델이 어려워하던 장거리 관계를 포착합니다.
🧩 비유로 이해 · Analogy
ENThink of reading a paragraph to answer a question: your eyes naturally linger on the few words that matter and skim the rest — attention does the same for a model.
KO질문에 답하려고 문단을 읽을 때, 눈은 중요한 몇몇 단어에 자연스럽게 머무르고 나머지는 훑어보게 됩니다. 어텐션도 모델에게 똑같은 일을 해 줍니다.
기본 용어 · Key terms
Query / Key / Value쿼리는 '무엇을 찾는가', 키는 '내가 무엇을 가졌는가', 값은 '실제로 전달할 정보'를 나타내는 세 벡터.
Softmax여러 점수를 합이 1인 확률 분포로 바꿔 주는 함수로, 어텐션 가중치를 만드는 데 쓰임.
Self-Attention같은 시퀀스 내의 토큰들이 서로를 참조하며 관계를 계산하는 어텐션 방식.
Multi-Head Attention여러 개의 어텐션을 병렬로 수행해 서로 다른 관점의 관계를 동시에 학습하는 구조.
Sparse Attention모든 토큰 쌍이 아니라 중요한 일부만 계산해 연산 비용을 줄이는 어텐션 기법.

오늘의 핵심 개념 · Key Concept

희소 어텐션 Sparse Attention
ENStandard attention compares every token with every other token, which costs computation that grows quadratically with sequence length.
KO표준 어텐션은 모든 토큰을 다른 모든 토큰과 비교하기 때문에, 연산량이 시퀀스 길이의 제곱에 비례해 증가합니다.
ENFor long inputs like high-resolution video, this quickly becomes the main bottleneck.
KO고해상도 비디오처럼 입력이 긴 경우, 이는 금세 가장 큰 병목이 됩니다.
ENSparse attention skips most low-importance comparisons and keeps only the blocks that carry meaningful signal.
KO희소 어텐션은 중요도가 낮은 대부분의 비교를 건너뛰고, 의미 있는 신호를 가진 블록만 유지합니다.
ENTop-p or Top-k routing is a common way to select which parts to keep at runtime.
KOTop-p 또는 Top-k 라우팅은 실행 중에 어떤 부분을 남길지 선택하는 흔한 방법입니다.
ENThe challenge is that different attention heads may keep different amounts of work, causing uneven load across GPUs.
KO문제는 어텐션 헤드마다 남기는 작업량이 달라, 여러 GPU 간 부하가 불균형해진다는 점입니다.
왜 지금? · Why now
ENToday's FVAttn paper tackles exactly this load-imbalance problem, showing why efficient attention is now a systems challenge, not just a modeling one.
KO오늘의 FVAttn 논문은 바로 이 부하 불균형 문제를 다루며, 효율적인 어텐션이 이제 모델링만이 아니라 시스템 차원의 과제임을 보여 줍니다.
