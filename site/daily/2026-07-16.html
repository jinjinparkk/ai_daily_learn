오늘의 기초 · Fundamentals

과적합과 정규화 Overfitting & Regularization
이게 뭐야? · What is it
ENOverfitting happens when a model memorizes the training data instead of learning general patterns.
KO과적합은 모델이 일반적인 패턴을 배우는 대신 학습 데이터를 암기해 버릴 때 발생합니다.
ENRegularization is a set of techniques that discourage a model from becoming too complex.
KO정규화는 모델이 지나치게 복잡해지지 않도록 억제하는 여러 기법을 말합니다.
ENTogether they describe the core tension in learning: fitting the data well while still generalizing to new data.
KO이 둘은 학습의 핵심 긴장을 설명합니다. 데이터를 잘 맞추면서도 새로운 데이터에 일반화하는 것입니다.
작동 원리 · How it works
ENDuring training, a model adjusts its parameters to reduce the error on the training set.
KO학습 중에 모델은 학습 세트의 오류를 줄이기 위해 파라미터를 조정합니다.
ENIf the model is very flexible, it can drive training error close to zero by capturing noise and quirks in the data.
KO모델이 매우 유연하면 데이터의 잡음과 특이점까지 포착해 학습 오류를 거의 0에 가깝게 줄일 수 있습니다.
ENBut that memorized noise does not appear in new data, so performance on unseen examples gets worse.
KO하지만 암기한 잡음은 새 데이터에 나타나지 않기 때문에, 처음 보는 예시에서의 성능은 나빠집니다.
ENRegularization fights this by adding a penalty for large or complex parameters, keeping the model simpler.
KO정규화는 크거나 복잡한 파라미터에 벌점을 부여해 모델을 더 단순하게 유지함으로써 이를 막습니다.
ENCommon methods include L1/L2 penalties, dropout, early stopping, and adding more diverse training data.
KO흔한 방법으로는 L1/L2 벌점, 드롭아웃, 조기 종료, 그리고 더 다양한 학습 데이터를 추가하는 것이 있습니다.
ENWe usually detect overfitting by comparing training accuracy with validation accuracy on held-out data.
KO우리는 보통 학습 정확도와 별도로 떼어 둔 검증 데이터의 정확도를 비교해 과적합을 감지합니다.
🧩 비유로 이해 · Analogy
ENIt is like a student who memorizes every answer from last year's exam but fails the new test, versus one who learns the concepts and can solve any question.
KO작년 시험의 답을 통째로 외웠지만 새 시험에는 떨어지는 학생과, 개념을 이해해 어떤 문제든 푸는 학생의 차이와 같습니다.

L2 penalty큰 가중치에 벌점을 주어 모델을 단순하게 만드는 정규화

오늘의 핵심 개념 · Key Concept

실행 범위 추정 Execution-Scope Estimation
ENExecution-scope estimation means judging how much effort a task really needs before starting it.
KO실행 범위 추정이란 작업을 시작하기 전에 그 작업에 실제로 얼마나 많은 노력이 필요한지를 판단하는 것입니다.
ENMany LLM agents waste resources by re-reading files and dependencies they have already seen.
KO많은 LLM 에이전트는 이미 본 파일과 의존성을 다시 읽으며 자원을 낭비합니다.
ENA one-line edit can turn into a full code-base audit when the agent assumes it needs maximum context.
KO에이전트가 최대한의 문맥이 필요하다고 가정하면, 한 줄 수정이 코드베이스 전체 감사로 커질 수 있습니다.
ENThe idea is to estimate a task's difficulty, execute a minimum viable path, and expand scope only if verification fails.
KO핵심은 작업 난이도를 추정하고, 최소한의 실행 경로를 수행한 뒤, 검증에 실패할 때만 범위를 넓히는 것입니다.
ENThis can cut cost and tokens dramatically while keeping the same success rate.
KO이렇게 하면 성공률은 유지하면서 비용과 토큰을 크게 줄일 수 있습니다.
왜 지금? · Why now
ENAs agents move into real workflows, paying for wasted effort adds up, so efficiency is becoming as important as accuracy.
KO에이전트가 실제 업무에 투입되면서 낭비된 노력의 비용이 쌓이기 때문에, 효율성이 정확성만큼 중요해지고 있습니다.

Q. 모델이 과적합하고 있다는 분명한 신호는 무엇입니까?
What is a clear sign that a model is overfitting?
✅ 과적합은 학습 성능은 뛰어나지만 처음 보는 검증 데이터에서 성능이 나쁜 형태로 나타납니다.
Overfitting shows up as strong training performance but poor performance on unseen validation data.
Q. E3 방법에서 에이전트는 언제 범위를 확장합니까?
In the E3 method, when does the agent expand its scope?
✅ E3는 먼저 최소 경로를 실행하고, 결과 검증이 실패할 때만 범위를 확장합니다.
E3 executes a minimal path first and only expands scope if verification of the result fails.
Q. '순차성 격차' 논문은 노이즈 제거 단계에 대해 어떤 결론을 내렸습니까?
What did the 'seriality gap' paper conclude about denoising steps?
✅ 논문은 노이즈 제거 단계가 모델의 백본을 넘어서는 순차 연산을 제공하지 않음을 증명합니다.
The paper proves that denoising steps do not provide serial computation beyond the model backbone.
