오늘의 기초 · Fundamentals

LLM 에이전트와 도구 사용 LLM Agents & Tool Use
이게 뭐야? · What is it
ENAn LLM agent is a language model that can plan actions, call external tools, and use the results to reach a goal, rather than just answering in one shot.
KOLLM 에이전트는 한 번에 답하는 것이 아니라, 행동을 계획하고 외부 도구를 호출하며 그 결과를 활용해 목표에 도달할 수 있는 언어 모델입니다.
EN'Tool use' means the model can trigger functions like web search, a calculator, a code runner, or a database query.
KO'도구 사용'이란 모델이 웹 검색, 계산기, 코드 실행기, 데이터베이스 조회 같은 기능을 실행할 수 있다는 의미입니다.
ENThis turns a static text predictor into a system that can act on the world and gather fresh information.
KO이를 통해 정적인 텍스트 예측기가 세상에 영향을 주고 최신 정보를 수집할 수 있는 시스템으로 바뀝니다.
작동 원리 · How it works
ENThe model first receives a goal and a list of available tools with descriptions of what each one does.
KO모델은 먼저 목표와, 각 도구가 무엇을 하는지 설명이 담�a긴 사용 가능한 도구 목록을 받습니다.
ENIt reasons about the task and decides whether it needs a tool or can answer directly.
KO모델은 작업에 대해 추론하고, 도구가 필요한지 아니면 바로 답할 수 있는지 판단합니다.
ENWhen a tool is needed, the model outputs a structured request, such as the tool's name and its arguments.
KO도구가 필요할 때 모델은 도구의 이름과 인자 같은 구조화된 요청을 출력합니다.
ENAn external system runs the tool and feeds the result back into the model as new context.
KO외부 시스템이 그 도구를 실행하고, 결과를 새로운 맥락으로 모델에 다시 전달합니다.
ENThe model repeats this loop of reasoning, acting, and observing until the task is complete.
KO모델은 추론하고, 행동하고, 관찰하는 이 순환을 작업이 끝날 때까지 반복합니다.
ENMemory can be added so the agent remembers earlier steps across a long task.
KO긴 작업 동안 에이전트가 이전 단계를 기억하도록 메모리를 추가할 수도 있습니다.
🧩 비유로 이해 · Analogy
ENThink of the agent as a smart assistant with a phone: it doesn't know everything, but it knows who to call and how to use apps to find the answer and finish the job.
KO에이전트를 스마트폰을 든 똑똑한 비서라고 생각해 보세요. 모든 것을 알지는 못하지만, 누구에게 연락하고 어떤 앱을 써야 답을 찾아 일을 끝낼 수 있는지 압니다.
기본 용어 · Key terms
Tool Calling (도구 호출)모델이 외부 함수나 API를 실행하도록 구조화된 요청을 생성하는 기능.
ReAct Loop (추론-행동 순환)추론(Reasoning)과 행동(Acting)을 번갈아 반복하며 문제를 푸는 에이전트 패턴.
MCP (Model Context Protocol)모델이 외부 도구와 데이터 소스에 표준화된 방식으로 연결하도록 돕는 프로토콜.
Long-Horizon Task (장기 과제)여러 단계를 거쳐야 완료되는, 긴 흐름의 복잡한 작업.
Episodic Memory (일화 기억)에이전트가 과거 상호작용과 사건을 저장해 두었다가 나중에 참고하는 기억 구조.

오늘의 핵심 개념 · Key Concept

생성형 월드 모델 Generative World Models
ENA world model learns to predict what a scene or environment will look like next, given the current state and an action.
KO월드 모델은 현재 상태와 어떤 행동이 주어졌을 때, 장면이나 환경이 다음에 어떻게 변할지 예측하는 법을 학습합니다.
ENGenerative world models create these future observations directly, often as video frames, instead of hand-coding every rule.
KO생성형 월드 모델은 모든 규칙을 일일이 코딩하는 대신, 이러한 미래 관측값을 주로 비디오 프레임 형태로 직접 생성합니다.
ENFor games, this means playable worlds can be synthesized on the fly as the user interacts with them.
KO게임에서는 사용자가 상호작용하는 즉시 플레이 가능한 세계가 실시간으로 만들어질 수 있음을 뜻합니다.
ENFor robots, richer signals like depth and motion make predictions closer to the physical actions a robot must take.
KO로봇에게는 깊이와 움직임 같은 더 풍부한 신호가 예측을 로봇이 실제로 취해야 할 물리적 행동에 더 가깝게 만듭니다.
ENThis lets teams train policies from generated data instead of only costly real-world demonstrations.
KO덕분에 값비싼 실제 시연에만 의존하지 않고, 생성된 데이터로 정책을 학습시킬 수 있습니다.
왜 지금? · Why now
ENSeveral of today's top papers build world models for both entertainment and robotics, signaling that this approach is maturing fast.
KO오늘 상위 논문 중 여러 편이 엔터테인먼트와 로보틱스 양쪽에서 월드 모델을 구축하고 있어, 이 접근법이 빠르게 성숙하고 있음을 보여줍니다.

핵심 영어 표현 · Vocabulary

tool use
도구 사용 (모델이 외부 함수·API를 실행하는 능력)
“Tool use lets the model fetch live data instead of guessing.”
도구 사용 덕분에 모델은 추측하는 대신 실시간 데이터를 가져올 수 있습니다.
autoregressive
자기회귀적인 (이전 출력을 바탕으로 다음 것을 하나씩 생성하는 방식)
“The model autoregressively generates each video frame from the previous state.”
이 모델은 이전 상태로부터 각 비디오 프레임을 자기회귀적으로 생성합니다.
long-horizon
장기 시야의 (많은 단계를 거쳐야 하는 긴 작업 관련)
“Training long-horizon agents is expensive because tasks span many turns.”
장기 에이전트 훈련은 작업이 여러 턴에 걸쳐 있어 비용이 큽니다.
extrapolate
외삽하다 (학습한 범위를 넘어서까지 일반화하다)
“HiLS can extrapolate to contexts 64 times longer than it was trained on.”
HiLS는 학습한 것보다 64배 긴 문맥까지 외삽할 수 있습니다.
open-weight
오픈 웨이트 (모델 가중치가 공개된)
“Gemma 4 is an open-weight model, so developers can run it locally.”
Gemma 4는 오픈 웨이트 모델이라 개발자가 로컬에서 실행할 수 있습니다.
