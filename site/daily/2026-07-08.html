오늘의 기초 · Fundamentals

LLM 에이전트와 도구 사용 LLM Agents & Tool Use
이게 뭐야? · What is it
ENAn LLM agent is a language model that can plan actions, call external tools, and use the results to reach a goal, rather than just producing one block of text.
KOLLM 에이전트는 단순히 한 덩어리의 텍스트만 생성하는 것이 아니라, 행동을 계획하고 외부 도구를 호출하며 그 결과를 활용해 목표에 도달하는 언어 모델입니다.
ENTool use means the model can trigger things like web search, a calculator, a code runner, or a database query when it needs help.
KO도구 사용이란 모델이 도움이 필요할 때 웹 검색, 계산기, 코드 실행기, 데이터베이스 질의 같은 것들을 실행할 수 있다는 뜻입니다.
ENThis turns a passive text generator into an active system that can take steps in the world.
KO이로써 수동적인 텍스트 생성기가 세상에서 실제로 단계를 밟아 나가는 능동적인 시스템으로 바뀝니다.
작동 원리 · How it works
ENFirst, the agent receives a goal and reasons about what it needs to do next.
KO먼저 에이전트는 목표를 받고 다음에 무엇을 해야 할지 추론합니다.
ENIt then decides whether to answer directly or to call a tool, choosing the right tool and its inputs.
KO그다음 직접 답할지 아니면 도구를 호출할지 결정하고, 적절한 도구와 그 입력값을 고릅니다.
ENThe tool runs and returns a result, such as a search snippet or a computed number, which is fed back into the model.
KO도구가 실행되어 검색 결과나 계산된 숫자 같은 결과를 반환하고, 이는 다시 모델에 입력됩니다.
ENThe agent observes this result, updates its plan, and may call another tool in a loop until the task is done.
KO에이전트는 이 결과를 관찰하고 계획을 갱신하며, 작업이 끝날 때까지 반복적으로 또 다른 도구를 호출할 수 있습니다.
ENThis observe-think-act cycle is often called a ReAct loop, and memory can be added so the agent remembers earlier steps.
KO이 관찰-사고-행동 순환은 흔히 ReAct 루프라고 불리며, 메모리를 추가하면 에이전트가 이전 단계를 기억할 수 있습니다.
ENStandards like MCP (Model Context Protocol) make it easier to connect many tools to an agent in a consistent way.
KOMCP(모델 컨텍스트 프로토콜) 같은 표준은 여러 도구를 일관된 방식으로 에이전트에 연결하기 쉽게 해줍니다.
🧩 비유로 이해 · Analogy
ENThink of the LLM as a smart office worker: instead of guessing every answer, they pick up the phone, open a spreadsheet, or search the archive, then combine those results to finish the job.
KOLLM을 유능한 사무직 직원이라고 생각해 보세요. 모든 답을 추측하는 대신 전화를 걸고, 스프레드시트를 열고, 자료실을 검색한 뒤 그 결과들을 종합해 일을 마무리하는 것과 같습니다.
기본 용어 · Key terms
Tool / Function Calling모델이 외부 기능(검색, 코드 실행 등)을 정해진 형식으로 호출하는 기능
ReAct loop추론(Reason)과 행동(Act)을 번갈아 반복하는 에이전트 실행 방식
MCP (Model Context Protocol)에이전트와 외부 도구·데이터를 표준 방식으로 연결하는 프로토콜
Long-horizon task여러 단계에 걸쳐 오래 진행되는 복잡한 작업
Episodic memory에이전트가 이전 상호작용이나 사건을 저장해 나중에 활용하는 기억

오늘의 핵심 개념 · Key Concept

생성형 월드 모델 Generative World Model
ENA world model learns to predict how an environment will change in response to actions.
KO월드 모델은 어떤 행동에 반응해 환경이 어떻게 변할지를 예측하도록 학습합니다.
ENA generative world model goes further by synthesizing the next observations, such as future video frames, directly.
KO생성형 월드 모델은 여기서 더 나아가 미래의 영상 프레임 같은 다음 관측을 직접 생성합니다.
ENIn games, this lets players interact with a world that is created on the fly instead of hand-built by developers.
KO게임에서는 개발자가 일일이 만든 세계 대신 즉석에서 생성되는 세계와 플레이어가 상호작용할 수 있게 됩니다.
ENIn robotics, predicting future states, depth, and motion narrows the gap between imagining an outcome and choosing a real action.
KO로보틱스에서는 미래 상태, 깊이, 움직임을 예측함으로써 결과를 상상하는 것과 실제 행동을 고르는 것 사이의 간극을 좁힙니다.
ENBecause collecting real robot data is slow, world models can act as simulators that generate training trajectories cheaply.
KO실제 로봇 데이터 수집은 느리기 때문에, 월드 모델은 훈련용 궤적을 저렴하게 생성하는 시뮬레이터 역할을 할 수 있습니다.
왜 지금? · Why now
ENSeveral of today's top papers—AlayaWorld, RynnWorld-4D, and RynnWorld-Teleop—all build on this idea, signaling that world models are becoming a major research frontier.
KO오늘 상위권 논문인 AlayaWorld, RynnWorld-4D, RynnWorld-Teleop이 모두 이 아이디어 위에 세워져 있어, 월드 모델이 주요 연구 최전선이 되고 있음을 보여줍니다.

Why does HiLS Attention aim to use sparse instead of dense attention?
✅ 밀집 어텐션은 제곱으로 비용이 커지고 긴 문맥으로의 외삽이 부실하기 때문에, HiLS는 선택된 청크에 주목하도록 학습합니다.
Dense attention scales quadratically and extrapolates poorly to longer contexts, so HiLS learns to attend to selected chunks instead.

핵심 영어 표현 · Vocabulary

autoregressive
자기회귀적인, 이전 출력을 바탕으로 다음 것을 순차 생성하는
“The model autoregressively generates future video frames one step at a time.”
그 모델은 미래 영상 프레임을 한 번에 한 단계씩 자기회귀적으로 생성합니다.
long-horizon
장기 지평의, 여러 단계에 걸쳐 오래 이어지는
“Training agents on long-horizon tasks remains computationally expensive.”
장기 지평 작업으로 에이전트를 훈련하는 것은 여전히 계산 비용이 큽니다.
extrapolate
외삽하다, 학습 범위를 넘어 확장 적용하다
“The method extrapolates beyond 64 times the training context length.”
그 방법은 학습 문맥 길이의 64배를 넘어 외삽합니다.
distillation
증류, 강한 모델의 지식을 더 작은 모델로 옮기는 기법
“On-policy distillation trains a student agent to match a stronger teacher.”
온폴리시 증류는 더 강한 교사를 따라 하도록 학생 에이전트를 훈련합니다.
encoder-free architecture
인코더 프리 구조, 별도 인코더 없이 원시 입력을 직접 처리하는 구조
“Gemma 4's 12B model uses a unified encoder-free architecture for audio and image patches.”
Gemma 4의 12B 모델은 오디오와 이미지 패치를 위해 통합된 인코더 프리 구조를 사용합니다.
throughput
처리량, 단위 시간당 처리되는 작업의 양
“Speculative decoding aims to improve inference throughput in serving systems.”
추측 디코딩은 서빙 시스템에서 추론 처리량을 개선하는 것을 목표로 합니다.
embodied
체화된, 물리적 몸이나 로봇을 통해 세상과 상호작용하는
“Embodied world models help robots anticipate how a scene moves under interaction.”
체화형 월드 모델은 로봇이 상호작용 시 장면이 어떻게 움직일지 예측하도록 돕습니다.
