오늘의 기초 · Fundamentals

경사하강법 Gradient Descent
이게 뭐야? · What is it
ENGradient descent is an algorithm that gradually adjusts a model's parameters to make its errors as small as possible.
KO경사하강법은 모델의 오차를 가능한 한 작게 만들기 위해 파라미터를 점진적으로 조정하는 알고리즘입니다.
ENIt is the core learning process behind neural networks, from small classifiers to giant language models.
KO작은 분류기부터 거대한 언어 모델까지, 신경망을 학습시키는 핵심 학습 과정입니다.
ENThe idea is simple: measure how wrong the model is, then take a small step in the direction that reduces that wrongness.
KO아이디어는 단순합니다. 모델이 얼마나 틀렸는지 측정한 뒤, 그 오차를 줄이는 방향으로 작은 한 걸음을 내딛는 것입니다.
작동 원리 · How it works
ENFirst, we define a loss function that gives a single number describing how far the model's predictions are from the correct answers.
KO먼저 모델의 예측이 정답에서 얼마나 벗어났는지를 하나의 숫자로 표현하는 손실 함수를 정의합니다.
ENNext, we compute the gradient, which tells us how the loss changes when we nudge each parameter a tiny bit.
KO다음으로 각 파라미터를 아주 조금 움직였을 때 손실이 어떻게 변하는지를 알려주는 기울기(그래디언트)를 계산합니다.
ENThe gradient points in the direction of steepest increase, so we move in the opposite direction to decrease the loss.
KO기울기는 손실이 가장 가파르게 증가하는 방향을 가리키므로, 손실을 줄이려면 그 반대 방향으로 움직입니다.
ENThe size of each step is controlled by the learning rate: too large and training becomes unstable, too small and it becomes painfully slow.
KO각 걸음의 크기는 학습률로 조절합니다. 너무 크면 학습이 불안정해지고, 너무 작으면 지나치게 느려집니다.
ENWe repeat this measure-and-step cycle many times until the loss stops meaningfully improving.
KO이 '측정하고 한 걸음 내딛기' 과정을 손실이 더 이상 의미 있게 개선되지 않을 때까지 여러 번 반복합니다.
ENIn practice we often use stochastic gradient descent, which updates parameters using small random batches of data for speed and efficiency.
KO실제로는 속도와 효율을 위해 작은 무작위 데이터 묶음으로 파라미터를 갱신하는 확률적 경사하강법을 자주 사용합니다.
🧩 비유로 이해 · Analogy
ENImagine hiking down a foggy mountain: you can't see the bottom, so you feel the slope under your feet and take a step downhill, again and again, until the ground flattens out.
KO안개 낀 산을 내려간다고 상상해 보세요. 바닥이 안 보이니 발밑의 경사를 느끼고 아래쪽으로 한 걸음씩 내딛기를 반복하다 보면, 땅이 평평해지는 지점에 도달하게 됩니다.
기본 용어 · Key terms
Loss function (손실 함수)모델의 예측이 정답과 얼마나 다른지를 하나의 숫자로 나타내는 함수.
Gradient (기울기/그래디언트)각 파라미터를 조금 바꿨을 때 손실이 얼마나 변하는지를 나타내는 방향과 크기.
Learning rate (학습률)한 번의 갱신에서 파라미터를 얼마나 크게 움직일지 정하는 값.
Stochastic Gradient Descent (확률적 경사하강법)전체 데이터 대신 작은 배치로 기울기를 추정해 빠르게 갱신하는 방식.
Convergence (수렴)반복 학습을 통해 손실이 더 이상 크게 줄지 않는 안정 상태에 도달하는 것.

오늘의 핵심 개념 · Key Concept

비디오 확산 모델 Video Diffusion Model
ENA diffusion model learns to generate data by gradually removing noise from a random starting point.
KO확산 모델은 무작위 시작점에서 잡음을 점진적으로 제거하며 데이터를 생성하는 법을 학습합니다.
ENDuring training, clean data is corrupted with noise, and the model learns to reverse that corruption step by step.
KO학습 시에는 깨끗한 데이터에 잡음을 더한 뒤, 모델이 그 손상을 한 단계씩 되돌리는 법을 배웁니다.
ENA video diffusion model extends this idea to sequences of frames, so it must keep motion smooth and consistent over time.
KO비디오 확산 모델은 이 아이디어를 프레임 시퀀스로 확장하기 때문에, 시간에 걸쳐 움직임을 부드럽고 일관되게 유지해야 합니다.
ENThe hardest challenge is long-horizon stability: small errors can accumulate and cause drift over long clips.
KO가장 어려운 과제는 장기 안정성입니다. 작은 오차가 누적되어 긴 영상에서 화면이 점점 어긋나는 표류가 생길 수 있습니다.
ENToday's papers tackle this with autoregressive rollouts, teacher-student distillation, and geometry-aware training.
KO오늘의 논문들은 이를 자기회귀적 롤아웃, 교사-학생 증류, 기하 인지 학습 등으로 해결하려 합니다.
왜 지금? · Why now
ENVideo diffusion is quickly becoming a shared backbone for many tasks, so understanding its strengths and failure modes is increasingly important.
KO비디오 확산 모델이 여러 작업의 공통 기반으로 빠르게 자리 잡고 있어, 그 강점과 실패 양상을 이해하는 것이 점점 더 중요해지고 있습니다.

긴 자기회귀 롤아웃은 종종 시간적 표류를 겪습니다.
distillation
증류, 교사 모델의 지식을 학생 모델로 옮기는 학습 방식
“Self-distillation lets a model teach itself using cleaner supervision.”
자기증류는 모델이 더 깨끗한 지도를 활용해 스스로를 가르치게 합니다.
quantization
양자화, 모델 가중치를 더 적은 비트로 표현해 압축하는 기법
“Quantization lets large models run on cheaper hardware.”
양자화는 대형 모델을 더 저렴한 하드웨어에서 돌릴 수 있게 합니다.
inference
추론, 학습된 모델을 실제로 실행해 결과를 얻는 과정
“Inference prices have dropped dramatically over the past year.”
추론 비용은 지난 한 해 동안 극적으로 떨어졌습니다.
autoregressive
자기회귀적, 이전에 생성한 결과를 바탕으로 다음을 생성하는 방식
“An autoregressive model conditions each chunk on its own previous output.”
자기회귀 모델은 각 조각을 자신이 이전에 생성한 출력에 의존해 만듭니다.
interpretability
해석가능성, 모델 내부가 왜 그렇게 작동하는지 이해하는 연구 분야
“Interpretability tools help us see what a model is really doing.”
해석가능성 도구는 모델이 실제로 무엇을 하는지 볼 수 있게 도와줍니다.
