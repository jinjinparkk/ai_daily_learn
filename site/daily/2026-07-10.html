오늘의 기초 · Fundamentals

평가와 벤치마크 Evaluation & Benchmarks
이게 뭐야? · What is it
ENEvaluation is the process of measuring how well an AI model performs on a task in a fair and repeatable way.
KO평가란 AI 모델이 어떤 작업을 얼마나 잘 수행하는지 공정하고 반복 가능한 방식으로 측정하는 과정입니다.
ENA benchmark is a standardized dataset and scoring rule that many models are tested on, so their results can be compared directly.
KO벤치마크는 여러 모델이 함께 테스트되는 표준화된 데이터셋과 채점 규칙으로, 그 결과를 서로 직접 비교할 수 있게 해 줍니다.
ENWithout good evaluation, claims that a model is 'better' are just opinions rather than evidence.
KO좋은 평가가 없다면 어떤 모델이 '더 낫다'는 주장은 증거가 아니라 단순한 의견에 불과합니다.
작동 원리 · How it works
ENFirst, you define a task clearly, such as answering factual questions or classifying an image.
KO먼저 사실 질문에 답하기나 이미지 분류하기처럼 작업을 명확하게 정의합니다.
ENNext, you collect a test set of examples that the model did not see during training, to check real generalization.
KO다음으로 모델이 학습 중에 보지 못한 예시들로 이루어진 테스트셋을 모아, 진짜 일반화 능력을 확인합니다.
ENThen you choose a metric, such as accuracy, F1 score, or perplexity, that turns model behavior into a comparable number.
KO그 후 정확도, F1 점수, 퍼플렉서티처럼 모델의 동작을 비교 가능한 숫자로 바꿔 주는 지표를 선택합니다.
ENYou run every model on the same data with the same rules, so differences come from the models and not the setup.
KO모든 모델을 동일한 데이터와 동일한 규칙으로 실행해, 차이가 실험 환경이 아니라 모델 자체에서 나오도록 합니다.
ENFinally, you interpret the scores carefully, watching for data leakage, overfitting to the benchmark, and gaps between the test and the real world.
KO마지막으로 데이터 유출, 벤치마크에 대한 과적합, 그리고 테스트와 실제 세계 사이의 간극을 주의하며 점수를 신중하게 해석합니다.
🧩 비유로 이해 · Analogy
ENA benchmark is like a standardized exam: it lets you compare students fairly, but a high score does not always mean someone will do well in real life.
KO벤치마크는 표준화된 시험과 같습니다. 학생들을 공정하게 비교할 수 있게 해 주지만, 높은 점수가 항상 실제 삶에서의 성공을 의미하지는 않습니다.
기본 용어 · Key terms
Test set (테스트셋)모델 성능을 평가하기 위해 따로 떼어 둔, 학습에 쓰이지 않은 데이터.
Metric (지표)정확도나 F1 점수처럼 모델 성능을 숫자로 나타내는 측정 기준.
Perplexity (퍼플렉서티)언어 모델이 다음 단어를 얼마나 잘 예측하는지를 나타내는 값으로, 낮을수록 좋음.
Overfitting to the benchmark (벤치마크 과적합)모델이 특정 벤치마크 문제에만 잘 맞춰져, 실제 상황에서는 성능이 떨어지는 현상.
Held-out data (홀드아웃 데이터)공정한 평가를 위해 학습에서 제외하고 남겨 둔 데이터.

오늘의 핵심 개념 · Key Concept

성찰 기반 에이전트 최적화 Reflection-based Agent Optimization
ENAn AI agent is a model that takes actions step by step to complete a long task, such as browsing tools and files.
KOAI 에이전트는 도구와 파일을 다루는 것처럼 긴 작업을 단계별로 수행하며 행동을 취하는 모델입니다.
ENWhen an agent fails, we can record its execution trace, which is the full sequence of steps it took.
KO에이전트가 실패하면, 그가 밟은 전체 단계의 순서인 실행 트레이스를 기록할 수 있습니다.
ENReflection-based optimization uses another language model to diagnose these failures and suggest better policies.
KO성찰 기반 최적화는 또 다른 언어 모델을 이용해 이러한 실패를 진단하고 더 나은 정책을 제안합니다.
ENThe problem is that real traces are noisy: they contain many irrelevant steps that hide the true root cause.
KO문제는 실제 트레이스가 잡음이 많다는 점입니다. 진짜 근본 원인을 가리는 관련 없는 단계가 많이 섞여 있기 때문입니다.
ENMethods like STRACE try to filter redundant traces and locate the exact step that caused the failure.
KOSTRACE 같은 방법들은 중복된 트레이스를 걸러내고 실패를 일으킨 정확한 단계를 찾아내려 합니다.
왜 지금? · Why now
ENAs inference costs collapse and agents run for hours on real work, learning efficiently from their failures becomes the main bottleneck.
KO추론 비용이 급락하고 에이전트가 실제 업무를 몇 시간씩 수행하게 되면서, 실패로부터 효율적으로 학습하는 일이 핵심 병목이 되고 있습니다.

핵심 영어 표현 · Vocabulary

benchmark
벤치마크, 모델을 비교하기 위한 표준 평가 기준
“The new model set a record on the reasoning benchmark.”
새 모델이 추론 벤치마크에서 기록을 세웠습니다.
perplexity
퍼플렉서티, 언어 모델의 예측 성능 지표(낮을수록 좋음)
“Lower perplexity usually means the model predicts text more confidently.”
퍼플렉서티가 낮다는 것은 보통 모델이 텍스트를 더 자신 있게 예측한다는 뜻입니다.
execution trace
실행 트레이스, 에이전트가 밟은 단계들의 전체 기록
“They analyzed the execution trace to find where the agent went wrong.”
그들은 에이전트가 어디서 잘못됐는지 찾기 위해 실행 트레이스를 분석했습니다.
root cause
근본 원인, 문제를 실제로 일으킨 핵심 요인
“Fixing the symptom is not enough; you must find the root cause.”
증상만 고쳐서는 부족하며, 근본 원인을 찾아야 합니다.
attributable
출처를 밝힐 수 있는, 어떤 근거에 귀속시킬 수 있는
“The model retrieves knowledge that is attributable to a specific source.”
그 모델은 특정 출처로 귀속시킬 수 있는 지식을 검색합니다.
inference cost
추론 비용, 모델을 실행해 결과를 얻는 데 드는 비용
“Falling inference cost makes it practical to run agents for hours.”
추론 비용이 떨어지면서 에이전트를 몇 시간씩 실행하는 것이 현실적으로 가능해졌습니다.
generalization
일반화, 학습하지 않은 새로운 데이터에서도 잘 작동하는 능력
“Good generalization means the model works on inputs it never saw during training.”
좋은 일반화란 모델이 학습 중 본 적 없는 입력에서도 잘 작동한다는 뜻입니다.
