오늘의 기초 · Fundamentals

토큰화 Tokenization
이게 뭐야? · What is it
ENTokenization is the process of breaking text into small units called tokens that a model can process.
KO토큰화는 텍스트를 모델이 처리할 수 있는 토큰이라는 작은 단위로 쪼개는 과정입니다.
ENA token can be a whole word, a piece of a word, or even a single character, depending on the method used.
KO토큰은 사용하는 방법에 따라 하나의 단어일 수도, 단어의 조각일 수도, 심지어 하나의 문자일 수도 있습니다.
ENLanguage models never see raw text directly; they only see the sequence of token IDs that tokenization produces.
KO언어 모델은 원래의 텍스트를 직접 보지 않으며, 토큰화가 만들어 낸 토큰 ID의 나열만을 봅니다.
작동 원리 · How it works
ENFirst, the tokenizer scans the input text and splits it into candidate pieces based on its rules.
KO먼저 토크나이저는 입력 텍스트를 훑으며 자신의 규칙에 따라 후보 조각들로 나눕니다.
ENModern models often use subword methods like Byte Pair Encoding (BPE), which merge frequent character pairs into larger units.
KO현대의 모델들은 흔히 자주 등장하는 문자 쌍을 더 큰 단위로 합치는 BPE(바이트 페어 인코딩) 같은 서브워드 방법을 사용합니다.
ENEach token is then mapped to a unique integer ID using a fixed vocabulary table.
KO그런 다음 각 토큰은 고정된 어휘 표를 이용해 고유한 정수 ID로 매핑됩니다.
ENThese IDs are fed into the model, which converts them into embedding vectors that carry meaning.
KO이 ID들은 모델에 입력되고, 모델은 이를 의미를 담은 임베딩 벡터로 변환합니다.
ENSubword tokenization lets a model handle rare or unseen words by combining smaller known pieces.
KO서브워드 토큰화는 알려진 작은 조각들을 결합함으로써 모델이 드물거나 처음 보는 단어도 다룰 수 있게 해 줍니다.
ENBecause pricing and context limits are counted in tokens, the number of tokens directly affects cost and speed.
KO요금과 문맥 길이 제한이 토큰 단위로 계산되기 때문에, 토큰 수는 비용과 속도에 직접적인 영향을 줍니다.
🧩 비유로 이해 · Analogy
ENThink of tokenization like cutting food before eating: a model cannot swallow a whole sentence at once, so it slices text into bite-sized pieces it can chew on.
KO토큰화는 음식을 먹기 전에 자르는 것과 같습니다. 모델은 문장 전체를 한 번에 삼킬 수 없으므로, 텍스트를 씹을 수 있는 한 입 크기의 조각으로 자릅니다.
기본 용어 · Key terms
Token (토큰)모델이 처리하는 텍스트의 기본 단위로, 단어·단어 조각·문자 등이 될 수 있습니다.
Vocabulary (어휘)토크나이저가 알고 있는 모든 토큰과 그에 대응하는 ID의 집합입니다.
Subword (서브워드)단어보다 작은 단위의 토큰으로, 드문 단어도 조합해 표현할 수 있게 합니다.
BPE (바이트 페어 인코딩)자주 함께 나타나는 문자 쌍을 반복적으로 합쳐 서브워드 어휘를 만드는 대표적 기법입니다.

오늘의 핵심 개념 · Key Concept

에이전트 검색 Agentic Search
ENAgentic search means a language model does not answer in one shot, but acts like an agent that issues multiple queries and reasons across turns.
KO에이전트 검색이란 언어 모델이 한 번에 답하지 않고, 여러 질의를 던지고 여러 차례에 걸쳐 추론하는 에이전트처럼 행동하는 것을 뜻합니다.
ENThis is powerful for hard questions where the answer must be assembled from several sources.
KO이는 답을 여러 출처에서 조합해야 하는 어려운 질문에 특히 강력합니다.
ENBut agents can get trapped in repetitive loops, wasting their search budget when a query keeps failing.
KO하지만 에이전트는 어떤 질의가 계속 실패할 때 반복적인 루프에 갇혀 검색 예산을 낭비할 수 있습니다.
ENOne paper today shows that a document useful for a single reader may be useless in an agent loop, and vice versa.
KO오늘의 한 논문은 단일 독자에게 유용한 문서가 에이전트 루프에서는 쓸모없을 수 있으며, 그 반대도 성립함을 보여 줍니다.
ENSuch 'bridge documents' matter not for what they say now, but for what they let the agent do next.
KO이러한 '브리지 문서'는 지금 무엇을 말하는지가 아니라, 에이전트가 다음에 무엇을 할 수 있게 하는지 때문에 중요합니다.
왜 지금? · Why now
ENAs tool-using LLM agents become mainstream, we need new ways to measure what really helps them succeed over many steps.
KO도구를 사용하는 LLM 에이전트가 주류가 되면서, 여러 단계에 걸친 성공에 실제로 도움이 되는 것이 무엇인지 측정하는 새로운 방법이 필요해졌습니다.
