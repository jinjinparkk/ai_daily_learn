오늘의 기초 · Fundamentals

LLM 에이전트와 도구 사용 LLM Agents & Tool Use
이게 뭐야? · What is it
ENAn LLM agent is a language model wrapped in a loop that lets it plan, take actions, and observe results instead of just answering once.
KOLLM 에이전트는 한 번만 답하는 것이 아니라 계획하고 행동하며 결과를 관찰할 수 있도록 반복 루프로 감싼 언어 모델입니다.
ENTool use means the agent can call external functions—like a search engine, a calculator, or a code runner—to get information it does not have.
KO도구 사용이란 에이전트가 검색 엔진, 계산기, 코드 실행기 같은 외부 기능을 호출해 스스로 가지지 못한 정보를 얻는 것을 뜻합니다.
ENTogether, these ideas turn a static text generator into a system that can complete multi-step tasks in the real world.
KO이 두 개념이 합쳐지면 정적인 텍스트 생성기가 현실 세계에서 여러 단계의 작업을 완수하는 시스템으로 바뀝니다.
작동 원리 · How it works
ENThe agent starts with a goal written in natural language, such as 'find the cheapest flight and book it.'
KO에이전트는 '가장 저렴한 항공편을 찾아 예약하라' 같은 자연어로 된 목표에서 시작합니다.
ENIt reasons about the next step, often writing out a short plan or a 'chain of thought.'
KO에이전트는 다음 단계를 추론하며, 흔히 짧은 계획이나 '사고 사슬(chain of thought)'을 작성합니다.
ENWhen it needs outside information, it emits a structured tool call that names the tool and its arguments.
KO외부 정보가 필요하면 도구 이름과 인자를 지정하는 구조화된 도구 호출을 생성합니다.
ENThe system runs that tool, captures the output, and feeds the result back into the model as a new observation.
KO시스템은 그 도구를 실행하고 출력을 받아, 그 결과를 새로운 관찰로서 모델에 다시 입력합니다.
ENThe model repeats this observe-think-act loop until it decides the task is finished or a stopping condition is met.
KO모델은 이 관찰-사고-행동 루프를 작업이 끝났다고 판단하거나 종료 조건이 충족될 때까지 반복합니다.
ENBecause each step consumes compute, well-designed agents also decide when to stop early to avoid wasting resources.
KO각 단계마다 연산이 소모되므로, 잘 설계된 에이전트는 자원 낭비를 막기 위해 언제 조기에 멈출지도 판단합니다.
🧩 비유로 이해 · Analogy
ENThink of an LLM agent like a chef who not only knows recipes but can open the fridge, taste the sauce, and adjust—checking ingredients (tools) and correcting course until the dish is done.
KOLLM 에이전트는 레시피만 아는 요리사가 아니라 냉장고를 열고 소스를 맛보며 조정하는 요리사와 같습니다. 재료(도구)를 확인하고 요리가 완성될 때까지 방향을 바로잡습니다.
기본 용어 · Key terms
Tool Call (도구 호출)에이전트가 외부 함수나 API를 실행하기 위해 도구 이름과 인자를 지정해 내보내는 구조화된 요청.
Chain of Thought (사고 사슬)모델이 최종 답에 도달하기 전에 중간 추론 단계를 명시적으로 써 내려가는 방식.
Agent Loop (에이전트 루프)관찰-사고-행동을 반복하며 작업을 진행하는 에이전트의 핵심 순환 구조.
Orchestration (오케스트레이션)여러 도구, 모델, 하위 에이전트의 실행 순서와 흐름을 조율하는 상위 제어 로직.
Early Abort (조기 중단)실패가 예상되는 작업 궤적을 미리 감지해 연산을 낭비하지 않도록 중간에 멈추는 기법.

오늘의 핵심 개념 · Key Concept

통합 멀티모달 생성 Unified Multimodal Generation
ENTraditionally, each vision task—detection, segmentation, depth—needed its own specialized model architecture and output head.
KO전통적으로 탐지, 분할, 깊이 추정 같은 각 시각 과제는 저마다 전용 모델 구조와 출력 헤드를 필요로 했습니다.
ENUnified multimodal generation reframes all these tasks as generating text, images, or a mix, inside one shared model.
KO통합 멀티모달 생성은 이 모든 과제를 하나의 공유 모델 안에서 텍스트, 이미지, 또는 그 혼합을 생성하는 문제로 재정의합니다.
ENThe task is specified with a natural-language instruction and optional visual prompts, rather than by changing the network itself.
KO과제는 네트워크 자체를 바꾸는 대신 자연어 지시문과 선택적 시각 프롬프트로 지정됩니다.
ENThis lets a single model match specialized systems across detection, OCR, segmentation, depth, and camera pose estimation.
KO덕분에 하나의 모델이 탐지, OCR, 분할, 깊이, 카메라 자세 추정 전반에서 전문화된 시스템에 필적할 수 있습니다.
ENThe payoff is scalability: new capabilities can be folded into a general foundation model without redesigning architecture.
KO이점은 확장성입니다. 구조를 다시 설계하지 않고도 새로운 기능을 범용 파운데이션 모델에 통합할 수 있습니다.
왜 지금? · Why now
ENAs foundation models grow, the field is shifting from many narrow tools toward single systems that absorb diverse tasks, which lowers engineering and maintenance costs.
KO파운데이션 모델이 커지면서, 분야는 좁은 도구 여러 개에서 다양한 과제를 흡수하는 단일 시스템으로 옮겨가고 있으며, 이는 엔지니어링과 유지보수 비용을 낮춥니다.

환각 (모델이 근거 없이 사실처럼 생성한 잘못된 내용)
“The verifier filters training data to prevent hallucination in the briefings.”
그 검증기는 브리핑에서 환각을 막기 위해 학습 데이터를 걸러냅니다.
generalization
일반화 (학습하지 않은 새로운 상황에서도 잘 작동하는 능력)
“The robot policy showed stronger generalization to out-of-distribution perturbations.”
그 로봇 정책은 분포 밖 교란에 대해 더 강한 일반화를 보였습니다.
inference compute
추론 연산 (모델이 답을 생성할 때 소모하는 계산 자원)
“Early abort saved up to 47% of inference compute.”
조기 중단은 추론 연산을 최대 47%까지 절약했습니다.
state-conditioned policy
상태 조건부 정책 (현재 상태에 따라 다음 행동을 선택하는 규칙)
“DynaKRAG learns a state-conditioned policy for choosing evidence operations.”
DynaKRAG는 증거 연산을 선택하기 위한 상태 조건부 정책을 학습합니다.
foundation model
파운데이션 모델 (대규모로 사전학습되어 여러 과제에 활용되는 기반 모델)
“Unified generation offers a route to fold vision skills into a foundation model.”
통합 생성은 비전 능력을 파운데이션 모델에 통합하는 길을 제시합니다.
